# Vendor Report Guidelines

## Purpose

These guidelines help ensure that monthly reports are parsed correctly and on time. They are about **structural stability**, not specific wording or branding. As long as the overall structure remains consistent, minor wording changes are fine.

---

## 1. Sheet / Tab Stability

- **Do not rename sheets** from month to month if they contain the same data.
- If a sheet must be renamed, keep recognizable keywords in the name so it can still be identified (e.g., a daily sheet should always contain "DLY" or "Daily").
- Do not split a single report into a different number of sheets without advance notice.

## 2. Column Consistency

- **Keep the same columns in the same order** across all sheets of the same type.
- Do not drop columns from one sheet while keeping them in another sheet of the same report.
- If a column is no longer needed, remove it from **all** sheets of that report type, not just one.
- New columns can be added at the end, but do not insert them in the middle of existing columns.

## 3. Header Row Stability

- Keep the header row at the same row number in every monthly report.
- The header text for key columns (e.g., date, impressions, spend) should remain the same or use recognizable keywords.

## 4. Section Boundaries (for Multi-Section Sheets)

- If a sheet contains multiple data sections (e.g., DOOH and Rideshare), keep the section header label in the same column.
- Do not insert extra rows (subtotals, notes, blank separators) between sections unless those rows are truly blank in the date column.
- If a totals row is needed, place it immediately after the section's data and ensure the date cell is empty.

## 5. Data Granularity

- Provide data at the agreed grain (usually daily).
- Each row should represent one day. Do not merge days or provide only weekly summaries unless explicitly agreed.

## 6. File Format

- Submit files in `.xlsx` format.
- One file per reporting period (one file per month).
- Filename can include the month and year, but the internal structure matters more than the filename.

## 7. What NOT to Worry About

- Exact filename wording.
- Extra summary sheets or pivot tables (as long as the raw daily data sheet is still present).
- Branding, colors, or formatting.
- Extra columns at the end of a sheet, as long as the core columns (date, impressions, spend) remain in their relative positions.

---

## Questions?

If a report structure must change, please provide a sample file in advance so the parser can be updated before the final report is due.
